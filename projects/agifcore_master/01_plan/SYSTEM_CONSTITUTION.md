# System Constitution

## Purpose

This document sets the standing rules for AGIFCore behavior across Phase 1 and later phases. It is the constitutional layer below the master plan and above execution detail.

## Core Rules

- AGIFCore must remain local-first.
- AGIFCore must not rely on hidden external models.
- AGIFCore must not depend on cloud correctness.
- AGIFCore must be replayable, reversible, auditable, governed, and fail-closed.
- The user is the final human authority.
- Program Governor is the highest project authority below the user.
- No agent may self-approve a phase or its own artifact.
- No agent may treat report text alone as proof.

## Phase 1 Application

Phase 1 uses this constitution to freeze the requirement pack, the human-thinking target, and the governance language that later phases must obey.

Phase 1 must keep the following in sync:

- `projects/agifcore_master/02_requirements/NON_NEGOTIABLES.md`
- `projects/agifcore_master/02_requirements/PHASE_APPROVAL_RULES.md`
- `projects/agifcore_master/01_plan/PHASE_GATE_CHECKLIST.md`
- `projects/agifcore_master/01_plan/HUMAN_THINKING_TARGET.md`

## Enforcement Notes

- Constitution Keeper guards this file and the requirement pack.
- Program Governor verifies that the constitution stays aligned with the frozen master plan.
- Any contradiction with the master plan or frozen phase rules must be treated as a blocker, not as a style issue.
