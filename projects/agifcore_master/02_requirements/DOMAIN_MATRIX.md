# Domain Matrix

## Locked Proof Domains

AGIFCore uses the eight frozen proof domains named in the master plan:

- `finance_document_workflows`
- `pos_store_operations`
- `procurement_work_order_processing`
- `claims_case_handling`
- `maintenance_diagnostics`
- `building_home_infrastructure_events`
- `planning_coordination_workflows`
- `compliance_support_triage`

## What This Matrix Must Capture

For each proof domain, the matrix must eventually record:

- the local baseline
- a fabric run without transfer
- a fabric run with transfer
- the governed conversation layer
- replayable traces
- laptop and mobile profile checks

## Phase 1 Rule

In Phase 1, the domain matrix is a governance target for later proof work. It must not be used to imply that the proof domains are already complete.
