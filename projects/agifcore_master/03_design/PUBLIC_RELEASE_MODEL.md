# Public Release Model

## Purpose

This file defines the first-pass public release and publication lane for AGIFCore.

## Frozen release surfaces

- release notes
- claims matrix
- paper-draft status note
- final evidence index
- public release package
- GitHub/public release asset flow
- tag/release flow
- paper/publication package
- public evidence-bundle alignment

## Release rules

- Nothing may enter the public release lane without matching evidence and approved claims.
- Public packaging may summarize proof; it may not invent proof.
- Release assets must stay aligned to the same audited bundle-integrity and trace-evidence truth used internally.
- Publication flow is downstream of validation, not a substitute for it.

## Dependency notes

- Final release mechanics depend on later product-runtime, evidence, audit, and closure work.
- This first-pass model freezes the release lane as an explicit design responsibility now so it cannot be bolted on at the end.
- Public packaging remains blocked until later phase approval and evidence generation exist.
